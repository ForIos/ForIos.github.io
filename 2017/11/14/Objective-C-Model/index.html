Objective-C 对象模型

Objective-C 对象模型是基于类建立的。在苹果开源库中 runtime中有Objective-C 对象模型的实现细节。

对象

在Objective-C ,每一个对象都是某个类的实例,且这个对象的isa指针指向它所属的类

打开刚下载的runtime工程,在objc-private.h中127行,可以找到对Objective-C中的对象定义struct objc_object。 Objective-C 其实就是这个结构体对象,isa为唯一私有成员变量。

在同一个文件的51-52行, 可以找到Class 和 id 类型的定义,分别是struct objc_class 和 struct objc_object 类型的指针。为什么id类型可以指向任何对象。因为objc_class : objc_object 。 其中objc_class 就是Objective-C中类的定义

类

1.对象的类描述了对象的数据:
(1)内存大小
(2)成员变量的类型
(3)对象的布局
2.对象的行为:
(1)对象的响应消息
(2)实现的实例方法

元类

Objective-C 中的类,也是对象 是某一个类的实例。这个类我们称为元类(metaclass)。
调用类方法,如[NSObject new] 其实就是 通过isa到元类中找是否存在该方法。
那元类也是对象吗？是的话那它又是什么类的实例呢？是的，没错，元类也是对象（元类对象），元类也是某个类的实例，这个类我们称之为根元类（root metaclass）。不过，有一点比较特殊，那就是所有的元类所属的类都是同一个根元类（当然根元类也是元类，所以它所属的类也是根元类，即它本身）。根元类指的就是根类的元类，具体来说就是根类 NSObject 对应的元类。

因此，理论上我们也可以给元类发送消息，但是 Objective-C 倾向于隐藏元类，不想让大家知道元类的存在。元类是为了保持 Objective-C 对象模型在设计上的完整性而引入的，比如用来保存类方法等，它主要是用来给编译器使用的。
